# Дизайн-система Qwopex

Единый источник истины по **дизайн-токенам** и **UI-компонентам**. Документ
описывает, что есть в репозитории сейчас, и как этим пользоваться при разработке
шелла и мини-аппов.

## Принципы

- **Zero-build, vanilla.** Никаких сборщиков, пре/постпроцессоров и фреймворков.
  Токены — нативные CSS Custom Properties, компоненты — нативные Web Components.
- **Строгий CSP** (`style-src 'self'`, `script-src 'self'`, без `'unsafe-inline'`).
  Отсюда два следствия:
  - инлайновые `<style>` запрещены — стили компонентов либо в [styles.css](styles.css),
    либо в shadow DOM через **constructable stylesheet** (`adoptedStyleSheets`),
    а не через `<style>`;
  - инлайновые обработчики (`onclick=…`) запрещены — только `addEventListener`.
- **Framework-agnostic.** Компоненты не знают о мини-аппах и наоборот; общение —
  через токены (стили) и Promise-API хелперов (поведение).

## 1. Токены

Определены в [styles.css](styles.css) в начале файла: тёмная тема — в `:root`
(по умолчанию), светлая — переопределяет переменные под `body.light`. Переключение
темы — `document.body.classList.toggle("light")` (см. `toggleTheme` в
[js/app.js](js/app.js)); выбор хранится в `localStorage["theme"]`.

> **Правило:** в стилях используйте только `var(--…)`. Литеральный цвет допустим
> лишь там, где он намеренно не зависит от темы (например, белый текст поверх
> акцентного фона) или пока для него нет токена.

### 1.1 Цвета

| Токен | Тёмная (`:root`) | Светлая (`body.light`) | Назначение |
|---|---|---|---|
| `--bg` | `#0f1115` | `#f4f5f7` | Фон страниц/экранов |
| `--surface` | `#1c1f26` | `#ffffff` | Карточки, шапки, панели |
| `--surface-2` | `#2c2f36` | `#eef1f7` | Вложенная поверхность: чипы, аватар, иконка приложения, поле поиска |
| `--surface-3` | `#3a3d45` | `#e4e6ea` | Приподнятая поверхность: ховер чипов/пунктов меню |
| `--border` | `#2c2c2e` | `#e4e6ea` | Разделители и рамки |
| `--text` | `#ffffff` | `#111111` | Основной текст |
| `--text-muted` | `#8e8e93` | `#6b7280` | Вторичный текст, подписи, плейсхолдеры |
| `--accent` | `#3b82f6` | `#3b82f6` | Акцент: активные чипы, бейджи, ссылки-действия |
| `--accent-soft` | `#7aa2ff` | `#7aa2ff` | Мягкий акцент: иконки, ховер-ссылки, фокус-рамка |

| `--success-soft` | `rgba(0,255,136,.14)` | `rgba(10,157,79,.16)` | Подложка success-бейджа (бонусы) |

*Пустая ячейка «Светлая» = токен в светлой теме не переопределяется (значение
наследуется из `:root`).*

**`*-soft`-семейство** — полупрозрачные подложки статусов в пару к насыщенному
цвету (`--success`/`--danger`/`--warning`). Используются как фон бейджа/ховера,
поверх которого идёт основной цвет.

### 1.2 Радиусы

| Токен | Значение | Назначение |
|---|---|---|
| `--radius` | `12px` | Карточки, диалоги, крупные элементы |
| `--radius-sm` | `8px` | Кнопки, мелкие контролы |

*(Скругления-«таблетки» — `border-radius: 999px` — задаются литералом, токена не
требуют.)*

### 1.3 Как добавлять токен

1. Объявите в `:root` со значением для тёмной темы; если в светлой цвет другой —
   переопределите в `body.light`. Совпадает — не дублируйте (наследуется).
2. Назовите по **смыслу**, а не по виду: `--field-bg`, а не `--dark-grey`.
3. Если значение совпадает с уже существующим токеном — используйте тот, не плодите
   синонимы.
4. Добавьте строку в таблицу выше.

## 2. Компоненты

Лёгкие Web Components в [js/ui/](js/ui/). У каждого есть Promise-/синглтон-хелпер —
**в прикладном коде пользуйтесь хелпером**, а не классом напрямую.

### Shadow DOM vs Light DOM

- **`qx-modal`** — **shadow DOM**: самодостаточен, иконок из спрайта не требует,
  токены наследуются сквозь границу shadow (CSS custom properties проходят).
  Стили — constructable stylesheet внутри модуля.
- **`qx-sheet`, `qx-toast`** — **light DOM**: их пунктам/иконкам нужны символы из
  общего SVG-спрайта (`<use href="#i-…">`), а ссылка на фрагмент `#id` **не
  резолвится сквозь shadow**. Поэтому стили этих двух — в [styles.css](styles.css)
  (классы `.qx-sheet*`, `.toast*`), а разметка — в light DOM.

### 2.1 `confirmDialog` — модальное подтверждение

Файл: [js/ui/qx-modal.js](js/ui/qx-modal.js). Замена нативному `confirm()`.

```js
import { confirmDialog } from "./ui/qx-modal.js";

const ok = await confirmDialog({
  title: "Удалить объявление?",
  message: "Это действие необратимо.",
  confirmText: "Удалить",
  cancelText: "Отмена",   // по умолчанию «Отмена»
  danger: true,           // красная кнопка подтверждения
});
if (ok) { /* подтвердил */ }
```

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `title` | string | `""` | Заголовок |
| `message` | string | `""` | Текст (пустой — строка скрывается) |
| `confirmText` | string | `"OK"` | Подпись кнопки подтверждения |
| `cancelText` | string | `"Отмена"` | Подпись кнопки отмены |
| `danger` | boolean | `false` | Красит кнопку подтверждения в `--danger` |

**Возвращает** `Promise<boolean>`: `true` — подтвердил; `false` — отмена, `Esc`
или клик по фону.

**Доступность:** `role="dialog"`, `aria-modal`, фокус-ловушка между кнопками,
автофокус на «подтвердить», возврат фокуса инициатору при закрытии.

### 2.2 `actionSheet` — меню действий (bottom-sheet)

Файл: [js/ui/qx-sheet.js](js/ui/qx-sheet.js). Выезжающее снизу модальное меню;
пункты переиспользуют класс `.menu-item`.

```js
import { actionSheet } from "./ui/qx-sheet.js";

const act = await actionSheet({
  title: "Объявление",
  items: [
    { id: "edit",   label: "Редактировать", icon: "#i-edit" },
    { id: "pin",    label: "Закрепить",     icon: "#i-pin-action" },
    { id: "delete", label: "Удалить", icon: "#i-trash", danger: true },
  ],
});
switch (act) { /* act === id выбранного пункта | null */ }
```

| Параметр | Тип | Описание |
|---|---|---|
| `title` | string | Заголовок листа (опц.) |
| `items` | Array | Пункты меню |
| `items[].id` | string | Возвращается при выборе пункта |
| `items[].label` | string | Подпись |
| `items[].icon` | string | ID символа из спрайта, напр. `"#i-edit"` (опц.) |
| `items[].danger` | boolean | Красит пункт в `--danger` (опц.) |

**Возвращает** `Promise<string|null>`: `id` выбранного пункта; `null` — `Esc`,
клик по фону или свайп вниз.

**Доступность/жесты:** `role="dialog"`, `aria-modal`, фокус-ловушка по пунктам,
автофокус на первом, свайп вниз по «ручке» (> 80px) закрывает.

### 2.3 `toast` — всплывающее уведомление

Файл: [js/ui/qx-toast.js](js/ui/qx-toast.js). Замена `alert()` для коротких
сообщений. Синглтон: повторный вызов переиспользует один элемент (тосты не копятся).

```js
import { toast } from "./ui/qx-toast.js";

toast("Сохранено", "success");
toast("Не удалось загрузить", "error");
toast("Черновик восстановлен");          // type по умолчанию "info"
```

В мини-аппах тот же тост доступен без импорта — через `ctx.toast(msg)` (см.
контракт мини-аппа в [ROADMAP.md](ROADMAP.md)).

| Аргумент | Тип | По умолчанию | Описание |
|---|---|---|---|
| `message` | string | — | Текст |
| `type` | `"info" \| "success" \| "error"` | `"info"` | Иконка и ARIA-режим |

Показ ~2.6 с, затем авто-скрытие. **Доступность:** `error` → `role="alert"` +
`aria-live="assertive"`; остальное → `role="status"` + `aria-live="polite"`.

## 3. Иконки

Общий **SVG-спрайт** инъектируется в документ один раз (см. `loadIcons` в
[js/app.js](js/app.js)) из [icons.svg](icons.svg). Иконка вставляется ссылкой на
символ:

```html
<svg class="icon"><use href="#i-search" /></svg>
```

Класс `.icon` задаёт размер `20×20` и `stroke: currentColor` — цвет наследуется от
текста элемента. В `qx-sheet`/`qx-toast` SVG собирается через `createElementNS`
(обычный `createElement("svg")` ломает `<use href>`).

**Доступные символы** (`id`): `i-alert`, `i-arrow-left`, `i-bold`, `i-chat`,
`i-check`, `i-close`, `i-edit`, `i-eraser`, `i-file`, `i-grid`, `i-image`,
`i-info`, `i-italic`, `i-list`, `i-list-ol`, `i-logo`, `i-logout`, `i-market`,
`i-moon`, `i-more`, `i-paperclip`, `i-pin`, `i-pin-action`, `i-plus`, `i-search`,
`i-send`, `i-strike`, `i-sun`, `i-telegram`, `i-trash`, `i-underline`, `i-user`,
`i-wallet`.

## 4. Как добавить компонент

1. Файл в [js/ui/](js/ui/), нативный `class extends HTMLElement` без зависимостей.
2. **Решите про DOM:** нужен спрайт-иконкой → light DOM + стили в `styles.css`;
   самодостаточен → shadow DOM + constructable stylesheet (никаких `<style>` из-за
   CSP).
3. Только токены `var(--…)` в стилях; с фолбэком в shadow DOM
   (`var(--surface, #1c1f26)`) на случай отсутствия токена.
4. Экспортируйте удобный хелпер (Promise для модальных, синглтон для тостов).
5. Доступность: `role`/`aria-*`, фокус-ловушка и возврат фокуса для модальных,
   закрытие по `Esc`.
6. Задокументируйте здесь.
