# Qwopex → Суперапп. Дорожная карта реализации

Документ описывает архитектуру каркаса (уже внедрён) и детальный план
дальнейшей реализации. Стек согласован: **Vanilla JS + дизайн-токены**,
zero-build, строгий CSP, framework-agnostic мини-аппы. Без React/MUI.

---

## 0. Что уже сделано (каркас + мини-аппы на демо-данных)

```
index.html        — шелл: общий SVG-спрайт + шапка + точка монтирования #view
manifest.json     — id, scope, shortcuts, SVG + PNG (192/512/maskable) иконки
icon.svg          — maskable-иконка (safe-zone 80%)
images/           — растровые иконки icon-192/512/maskable-512.png
sw.js             — network-first навигация + SWR статика + network-only API
styles.css        — дизайн-токены (:root / body.light) + стили шелла/мини-аппов
js/
  app.js          — шелл: тема, лаунчер, хэш-роутинг, лайфцикл мини-аппов,
                    тосты, регистрация и обновление SW (плашка «Обновить»)
  sdk.js          — общий слой: утилиты, санитайзер, storage, шина, auth
  registry.js     — реестр мини-аппов (ленивый import)
  ui/
    qx-modal.js   — модалка/диалог подтверждения (confirmDialog, shadow DOM)
    qx-sheet.js   — bottom-sheet/меню действий (actionSheet, light DOM)
    qx-toast.js   — всплывающее уведомление (toast-синглтон, light DOM)
  data/           — демо-сторы поверх localStorage (до миграции на API):
    ads-store.js      — объявления (+ pending-фильтр, черновик)
    chats-store.js    — диалоги, непрочитанные, сводка по объявлению
    wallet-store.js   — баланс/бонусы/операции, pay() и topUp()
    categories.js     — единая таксономия категорий
  apps/
    ads.js        — «Объявления»: список, фильтры (категория/статус/поиск),
                    создание/редактирование, мультифото, статусы, чат-связь
    wallet.js     — «Кошелёк»: баланс, бонусы, история операций, пополнение
    chats.js      — «Чаты»: диалоги, вложения, непрочитанные, привязка к объяв.
    market.js     — «Маркет»: каталог, категории, карточка, оплата из Кошелька
```

> Мини-аппы работают на **демо-данных в localStorage** — UI/логика готовы,
> остаётся подключить бэкенд (Этапы 3–4) и realtime/платежи (Этапы 5–6).

**Легенда статусов ниже:** `[x]` — сделано, `[~]` — частично (готов фронт/демо,
ждёт бэкенд), `[ ]` — ещё не начато.

**Контракт мини-аппа** (`default export`):

```js
export default {
  mount(root, ctx) {},   // отрисовать UI в root
  unmount() {},          // снять слушатели/таймеры, очистить root
  onRoute(subpath) {},   // (опц.) реакция на под-маршрут без ремоунта
};
```

`ctx` = `{ user, subpath, bus, navigate(hash), toast(msg), scrollTop() }`.

**Маршрутизация:** хэш-based — `#/` лаунчер, `#/ads`, `#/ads/create`,
`#/wallet`, `#/chats`, `#/market`. Выбрано вместо History API, т.к. не
требует серверных rewrite-правил для статики.

## Принятые архитектурные решения

| Решение | Почему |
|---|---|
| Vanilla + токены, без React/MUI | zero-build, строгий CSP без `unsafe-inline`, минимальный вес, framework-agnostic мини-аппы |
| Хэш-роутинг | не нужны серверные rewrite-правила для статического PWA |
| Мини-аппы как ленивые ES-модули | загрузка только при открытии, изоляция, независимый релиз |
| SW: network-first навигация | пользователь всегда получает свежий shell, офлайн — фолбэк из кэша |
| SW: SWR для статики | мгновенный отклик + фоновое обновление ассетов |
| SW: без авто-`skipWaiting` | обновление по согласию (плашка), чтобы не смешать версии модулей |
| `connect-src 'self'` пока | бэкенда нет; расширим точечно при интеграции API |

## Риски

- **Один SW на все мини-аппы**: ошибка в стратегии кэширования затрагивает
  всю платформу — покрыть тестами, версионировать.
- **`document.execCommand` в редакторе** устарел — со временем заменить на
  Selection/Range API или contenteditable-обёртку.
- **localStorage как временное хранилище** объявлений — это только дев-режим,
  до Этапа 4 (миграция на API).
